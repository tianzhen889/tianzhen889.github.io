强大免费的内网穿透工具loophole
内网穿透 2023/11/10

最近在浏览YouTube时偶然发现一款强大的内网穿透工具：loophole，要知道，目前市面上号称免费的内网穿透工具大部分有以下几个缺点：

  • 限制带宽和流量
  • 不能指定二级域名或者不能免费指定
  • 不断变化的域名（不能在内网建站，只能短时间使用）
  • 需要公网ip做转发（都有公网ip了我还折腾什么）
  • 需要开启vpn，不能暴露公网，只能自己组局域网使用（zerotier之类）

但是，loophole就不存在以上问题。

1、什么是loophole

Loophole.cloud是若干免费内网穿透工具中提供公共服务且相对稳定的一个，支持Windows、Linux（包括嵌入式设备）、MacOS，同时提供cli和desktop程序。目前，loophole服务仅在欧洲托管。loophole的客户端cli是开源的，官方自述“就目前而言，我们希望通过测试期以稳定系统，希望明年能提供自托管服务”。loophole服务提供的功能包括：

  • 公开本地HTTP服务器
  • 公开在网络中任何计算机上运行的HTTP服务器
  • 通过HTTPS公开本地目录
  • 通过WebDav公开本地目录
  • 基本身份验证

功能上公开本地HTTP服务器，就是网站发布，这个网站可以是一个博客、一个影视库，也可以是一个论坛系统等。另外一款工具zrok将这类功能总结为共享文件，定义很准确，无论是html、css、js还是其他文件，无非都是公布出去别人能获取到并在浏览器或其他程序中解析、渲染、呈现。因此，webdav公开本地目录、前端调试等都是以为网站发布/文件共享功能为基础的。

2、安装

无需安装，只需要解压对应的压缩包即可，这里引用作者的Cli版本的压缩包链接：

Windows

Linux

MacOS

Linux或者MacOS也可以直接拉取：

wget https://github.com/loophole/cli/releases/download/1.0.0-beta.15/loophole-cli_1.0.0-beta.15_linux_armv7.tar.gz
#github无法下载时可使用以下备用链接
#wget https://appscross.com/Sharetools/Tools%20Share/loophole/cli/linux/loophole-cli_1.0.0-beta.15_linux_armv7.tar.gz

#解压并进入loophole目录
tar zxvf loophole-cli_1.0.0-beta.15_linux_armv7.tar.gz && cd loophole-cli_1.0.0-beta.15_linux_armv7/

3、网站发布

然后我们在命令行下打开，在发布内网网站之前，我们需要先登录，以Windows为例：

loophole account login

然后会给我们一个链接和验证码，只需打开验证即可，然后会要求我们登录，可以使用GitHub账号直接登录，之后返回cmd，我们可以通过指定端口和ip来映射到公网，或者开启https：

loophole http [端口] [ip] [--https]

可以映射局域网内的所有设备，只需端口和ip即可，十分方便。

在默认情况下，loophole会给我们分配一个随机的二级域名，在我们退出之前，这个二级域名不会发生变化。

如果想要自定义二级域名，只需在后面加上–hostname [自定义]，例如：

loophole http [端口] [ip] [--https] [--hostname][自定义]

如果自定义的域名没有被占用的话，loophole会为我们分配一个自定义.loophole.site的域名，这时我们就可以在公网访问这个域名了。而且虽然服务器在欧洲，实测不需要科学也能访问。

请注意，如果报错，请尝试更换一个域名

当然了，loophole还有其他玩法，后面我会继续分享。

坚定不移走适合自己的路线
个人纪录 2023/11/8

科班出身的人总会有一些各种各样的疑虑，我到底是走开发呢还是研究算法呢，我是拼绩点呢还是拿证书呢。也可能都没有，但是你总要选择，总要选择一条适合自己的路线进步。进入大学之后，诚然，很多人不是一开始就好好学习的，很多人前两年并没有积累什么技术。我也是这样，我也很后悔大一大二没有参加比赛，没有拿到什么证书。那我以后就要继续摆烂吗，我后面该怎么发展。

我相信很多人跟我一样的经历，学校本身没有竞赛氛围，专业老师也只会讲一些过时内容。唯一能学习的方式也就是通过b站，但是这没办法，如果选择了b站自学，如果选择了走开发路线，那就一条路走到底，我一直觉得要做就做好，不做就不做。

所以如果你大一大二就没什么竞赛经历，算法水平不强，那你就别想着通过打比赛来得到认可，那不是你的路，你应该认清自己的优势，然后坚定不移走自己的道路，就算最后还是失败，至少学到了真东西。而不是人云亦云，到最后什么也没弄好。

当然，我是针对大一大二没有好好学习的同学，是时候做出选择了，如果是刚入学的同学，我还是建议不要给自己设上限，大数据，ai，前端后端，游戏开发都可以学的，技多不压身。

我只是希望无数想我这样的b站自学者，最后都能学有所成，进入大厂，实现自己的愿望。

标签页小图标（favicon）显示异常的问题
前端网页 2023/11/7

我在搭建静态网页时，发现favicon不能正常加载的问题，我去看了作者的源码，发现配置文件中没有favicon的接口，但是其教程里却有：

# Avatar image
avatar: /images/avatar.jpg

# Favicon
favicon: /images/favicon.png

# Home page background image
background:
    - /images/background.jpg

# Loading image
loading: /images/loading.gif

# Optional colors for category and tag
colors:
    - "#ffa2c4"
    - "#00bcd4"
    - "#03a9f4"
    - "#00a596"
    - "#ff7d73"

我以为不能正常加载是由于作者漏写了这个接口，我在加上后仍不能正常加载favicon，我研究了好久，发现作者其实是把favicon和avatar共用了一个图标，但这并不能解决我的问题，最后想到可能是由于标签内内容过多，然后我将layout.ejs中的移到了最前面，为了先渲染标签页图标，最后，我解决了问题。如下：

</head>
    <link rel="icon" href="<%- url_for(theme.avatar) %>" />

    <script src="/js/<file>"></script>
    <link rel="stylesheet" href="/css/<file>" />
    <canvas id="fireworks" style="position: fixed; top: 0; left: 0; width: 100vw; height: 100vh; pointer-events: none; z-index: 32767"></canvas>
    <script src="https://cdn.staticfile.org/animejs/3.2.1/anime.min.js"></script>
    <script src="/js/fireworks.min.js"></script>
    <canvas id="background" style="position: fixed; top: 0; left: 0; width: 100vw; height: 100vh; pointer-events: none; z-index: -1"></canvas>
    <script src="/js/background.min.js"></script>
    <div id="cursor"></div>
    <link rel="stylesheet" href="/css/cursor.min.css">
    <script src="/js/cursor.min.js"></script>
    <meta charset="utf-8" />
    <title><%= title %></title>
    <meta name="author" content="<%- config.author %>" />
    <meta name="description" content="<%- config.description %>" />
    <meta name="keywords" content="<%- config.keywords %>" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0" />
    
    <%- partial("import", { type }) %>
</head>
